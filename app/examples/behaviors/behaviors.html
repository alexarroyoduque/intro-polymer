Comportamientos

Polymer permite programar la lógica de un componente una vez y que otros elementos puedan usarla. De este modo se pueden heredar comportamientos comunes fácilmente evitando duplicidades de código.

No obstante si un componente que hereda un comportamiento necesita adaptar alguna de las funcionalidades proporcionadas es posible sobreescribirlas desde el propio componente.

Ejemplo

Todos los heroes pueden salvar el mundo y tienen un color favorito. Flash a diferencia de los otros heroes ha sobreescrito su comportamiento base para elegir otro color diferente.

intro-hero-behavior.html

  <script>
  HeroBehavior = {
      properties: {
          favouriteColor: String,
          times: {
              type: Number,
              value: 0
          }
      },
      saveTheWorld: function () {
          this.times++;
      },
      chooseFavouriteColor: function () {
          this.favouriteColor = 'green';
      }
  };
  </script>

intro-flash.html

<link rel="import"
    href="../../bower_components/polymer/polymer.html">

<link rel="import" href="intro-hero-behavior.html">

<dom-module id="intro-flash">

  <template>
    <p><b>Flash</b> ha salvado el mundo: <span>{{times}}</span> veces</p>
    <p><button on-click="saveTheWorld">Salva el mundo</button></p>
    <p>
      <button on-click="chooseFavouriteColor">Dime tu color favorito</button>
      <span>{{favouriteColor}}</span>
    </p>
  </template>

  <script>
    Polymer({
      is: "intro-flash",
      behaviors: [HeroBehavior],
      chooseFavouriteColor: function () {
        this.favouriteColor = 'red';
      }
    });
  </script>

</dom-module>
